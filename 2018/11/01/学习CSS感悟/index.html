知易行难

学习CSS感悟
Publish: 2018/11/1   
  • CSS

​ 我想要达到的目标，我要能实现任何我想实现的布局，我能能实现我看的所有布局。刚接触到css的时候，我知道这是给网页实现样式的，然网页变的好看的。刚开始写静态网页时，我始终记住一句话，
html是用来展现文档的结构的，然后关于样式的都归css。
当我开始学习写网页时，我在调样式时，经常会出现div不够用，然后就开始加div，然后又感觉多了，总之就很乱，然后就脑壳疼。。。
如果能勉强实现，我只有一秒的满足感。为什么呢，还是感觉乱。我不知道我能不能在实现。
我想到的方法是应该先分析。先确定文档的结构。在实现。

​ 我之前好像脑袋里也有这种想法但总是等不及或者没有养成这种习惯，分析是一件很累的事情，尤其是你不熟悉的事情，我会不知觉的规避这种想法。这个应该是我学生时代的毛病。我好像知道该怎么做，但是有畏难情绪。也是脑袋不聪明，却总想用“省力”的办法去做。还是“出来混总是要还的”。

css的具体内容我记得第一点是position的使用，然后接触到盒模型。然后我感觉我能做任何我想做的布局了。结果发现应该是刚刚进坑而已。在接着我发现伪元素我决定好高级哦：hover，鼠标放上去可以交互，很有意思。我现在能记得的就是line-height的使用还可以让元素居中。

学css的终极奥义还是 卖油翁的名言 “无他，唯手熟尔”. + 正确的方法（分析）
