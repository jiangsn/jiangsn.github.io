HTTP学习记录
Publish: 2018/10/28   
  • HTTP

HTTP协议的作用就是浏览器和服务器的之间通信的标准。（就知道理解双方的意思）

请求部分

浏览器发起请求的 ，服务器响应请求。
请求分为四部分
第一部分：请求的方法 如（GET POST DELETE .）等常用的GET 和 POST 和HTTP协议的版本
第二部分：以键值对的形式，请求的地址（域名），所使用的代理（也就是用什么去发起请求的），可以接受的内容，字符接 etc
第三部分：就是一个换行 。目的是为了第二部分和第四部分（始终存在）
第四部分：传给服务器的数据（如果有需要）

响应部分

服务器响应部分也分四部分
第一部分：协议/版本号 状态码 状态解释
第二部分：以键值对的形式，响应数据的描述。Content-Length （内容长度），Content-Type（内容类型）
第三部分：就是一个换行 。目的是为了第二部分和第四部分（始终存在）
第四部分：页面所展示的内容

用Chrome查看

  • 打开 Network
  • 输入网址
  • 选中第一个响应
  • 查看 Response Headers，点击「view source」，点击「view source」，点击「view source」
  • 你会看到响应的前两部分
  • 查看 Response 或者 Preview，你会看到响应的第 4 部分

在命令行下工作的文件传输工具。

curl -s -v -H "Frank: xxx" -- "https://www.baidu.com"
  • -s Silent or quiet mode. Don’t show progress meter or error messages. Makes Curl mute.
  • -v 输出更多信息
  • -H 为请求头添加信息（以键值对的形式）
